# 📖 사용자 지시 가이드

> **이 문서는 AI 에이전트 팀에게 아이디어를 어떻게, 어떤 수준으로 전달해야 하는지 안내합니다.**

---

## 🎯 핵심 원칙

AI 에이전트 팀은 **당신의 아이디어를 받아 자동으로 기획 → 설계 → 구현 → 검증**합니다.
하지만 입력의 품질이 출력의 품질을 결정합니다.

> **좋은 지시 = 명확한 목적 + 구체적 요구 + 제약조건**

---

## 📊 지시 레벨 가이드

### Level 1: 한 줄 아이디어 (최소)
가장 간단한 형태입니다. 시스템이 세부사항을 자동으로 추론합니다.

```
🎯 아이디어 > 할 일 관리 앱을 만들어줘
```

**시스템 동작:**
- PM이 "할 일 관리 앱"을 분석하여 기본 기능을 추론
- Planner가 일반적인 ToDo 앱 PRD 생성
- 세부사항은 시스템이 합리적으로 가정

**적합한 상황:** 빠른 프로토타입, 아이디어 탐색, 일반적인 기능의 프로젝트

---

### Level 2: 구체적 기능 포함 (권장)
핵심 기능과 대상 사용자를 명시합니다.

```
🎯 아이디어 > 프리랜서를 위한 프로젝트 관리 웹앱을 만들어줘. 
              클라이언트별 프로젝트 추적, 작업 시간 기록, 
              인보이스 자동 생성 기능이 필요해.
```

**시스템 동작:**
- PM이 "웹앱" 키워드 감지 → Designer 포함 파이프라인 선택
- Planner가 명시된 3가지 기능을 중심으로 PRD 작성
- Architect가 웹 프레임워크 기반 설계

**적합한 상황:** 대부분의 프로젝트, 명확한 목적이 있을 때

---

### Level 3: 상세 요구사항 (최대)
기술 스택, 제약조건, 비기능 요구사항까지 포함합니다.

```
🎯 아이디어 > Python FastAPI 기반의 REST API 서버를 만들어줘.
              - 사용자 인증 (JWT 토큰 방식)
              - 게시글 CRUD (제목, 내용, 작성일, 태그)
              - SQLite 데이터베이스 사용
              - 응답 시간 200ms 이내
              - 테스트 커버리지 80% 이상
              - Docker 배포 가능
```

**시스템 동작:**
- PM이 상세 요구사항을 그대로 전달
- Planner가 명시된 기능을 P0으로 분류한 PRD 작성
- Architect가 FastAPI + SQLite 기반 구체적 설계서 작성
- Coder가 설계서에 따라 정밀하게 구현

**적합한 상황:** 프로덕션 수준의 프로젝트, 기술적 요구가 명확할 때

---

## ✅ 좋은 지시 vs ❌ 나쁜 지시

### 좋은 지시 예시

| 지시 | 왜 좋은가? |
|------|-----------|
| "학생들의 출석을 QR코드로 체크할 수 있는 웹앱" | 대상(학생), 기능(출석 체크), 방법(QR코드), 플랫폼(웹앱) 명확 |
| "매일 오전 9시에 네이버 뉴스 헤드라인을 크롤링해서 슬랙으로 보내는 자동화 스크립트" | 시간(오전 9시), 소스(네이버), 출력(슬랙), 유형(자동화) 명확 |
| "GPT API를 사용해서 사용자가 입력한 한국어를 영어/일어/중국어로 번역해주는 CLI 도구" | API(GPT), 입력(한국어), 출력(3개 언어), 인터페이스(CLI) 명확 |

### 나쁜 지시 예시

| 지시 | 왜 나쁜가? | 개선 방법 |
|------|-----------|-----------|
| "뭔가 좋은 거 만들어줘" | 목적,기능,대상 모두 불명확 | 구체적인 아이디어 제시 |
| "앱 만들어" | 어떤 앱인지 전혀 알 수 없음 | "XX를 위한 YY 앱" 형태로 |
| "인스타그램 클론" | 범위가 너무 넓음 | MVP 범위를 좁혀서 제시 |

## 📝 지시 작성 체크리스트

지시를 작성할 때 아래 항목을 고려하면 좋습니다:

```
☐ 무엇을 만드는가?        (예: 할 일 관리 앱)
☐ 누구를 위한 것인가?      (예: 프리랜서)
☐ 핵심 기능은?            (예: 프로젝트 추적, 시간 기록)
☐ 어디서 실행되는가?       (예: 웹, 데스크톱, CLI)
☐ 특별한 기술 제약이 있는가? (예: Python만 사용, SQLite)
☐ 어떤 수준의 결과를 원하는가? (예: 프로토타입, 프로덕션)
```

> 모든 항목을 채울 필요는 없습니다. 2~3개만 명확해도 시스템이 잘 동작합니다.

## 🔄 대화형 모드에서의 흐름

```
1. 시스템 실행: python main.py
2. 아이디어 입력: "할 일 관리 웹앱을 만들어줘"
3. 시스템이 자동으로 처리:
   PM → Planner → Architect → Designer → Coder → Reviewer → Tester
4. 최종 보고서 확인
5. 새 아이디어 입력 또는 'q'로 종료
```
